## Track description (sponsor's words)

Most fans watch with a phone in hand. TxLINE gives live scores, real-time odds, match events for all 104 games — data only big operators had until now. "The experiences that could exist here have not been built yet."

Sponsor starter ideas: Group Sweepstake (live leaderboard) · AI Pundit Bot (Telegram alerts on goals/cards/odds shifts) · Hi-Lo Stats Game (streaks across 104 games).

## Judging criteria (build against these)

1. **Fan Accessibility & UX** — would a mainstream, non-technical fan open it regularly? Polish matters.
2. **Real-Time Responsiveness** — updates fluidly with what's happening on the pitch.
3. **Originality & Value Creation** — genuinely new fan interaction model, not a repackaged feed.
4. **Commercial & Monetization Path** — clear product utility / viable business model.
5. **Completeness & Execution** — feels like a finished end-to-end product feature; small deliberate scope is fine.
- **Demo video weighs heaviest** — judges review after the tournament ends; the video carries the entire experience.

## Hard requirements

- Live (mainnet or devnet) product **that works during a match**.
- **Must use TxLINE data as a live input AND sign up through Solana** (wallet-based sign-in).
- Demo video ≤ 5 min (Loom/YouTube): problem → live walkthrough → how TxLINE powers backend. Absolute screening requirement.
- Public GitHub repo.
- Working deployed link OR functional API endpoint judges can test at zero cost.
- Brief technical doc: core idea, highlights, list of TxLINE endpoints used.
- Feedback section: API experience, likes, friction.
- Functional product only — decks/wireframes/mockups auto-disqualified.
- Teams ≤ 3; AI agents allowed but submission owned by real person/entity eligible on Earn.

## Implications for idea-01 build

- **UX quality is judging criterion #1** — frontend polish gets real budget, not an afterthought.
- **Wallet sign-in required** (criterion: "sign up through Solana") — connect-wallet flow is mandatory anyway for pledges.
- Real-time board/feed = criterion #2 — SSE-driven live updates must be visible in the demo.
- Monetization slide for video: protocol fee on released pledges / brand-sponsored DAOs / white-label for causes.
- The on-chain settlement engine is our differentiator vs other fan apps (criterion #3 originality), but the video must lead with the fan story, not the cryptography.
